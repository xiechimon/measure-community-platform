# Java 后端需求基线

> 本文档是 Java 后端需求的正式基线，统一定义范围、排除边界、责任归属、需求编号、能力、实现状态及验收证据。功能范围唯一来源为《开发计划7.13(1).xlsx》的「阶段1」工作表。

## 范围

- 纳入 Excel「阶段1」第 3–188 行中可由 Java 后端承接的业务数据、领域流程、查询聚合、通知、审计、附件、导入导出、支付、核销、积分、设备接入与告警、第三方跳转统计，以及支撑居民侧小程序和物业侧移动端的 API。
- 后端实现基线为 Java 17 + Spring Boot；不新增其他后端技术栈。页面布局、视觉样式、按钮文案及其他纯前端事项不作为后端需求。
- Excel 已纳入且处理个人信息、支付、文件或外部接入的能力，后续可由《详细功能设计说明书-数智化社区服务平台-V7 版本.docx》补充字段、安全、隐私、可用性和接口约束；该补充不得扩张业务域。
- 覆盖矩阵的候选来源集合为本文件“来源覆盖矩阵：纳入行集合”列出的全部行。后续任务必须为其中每一行映射一个或多个稳定需求编号；行号未映射即为基线审查失败。

## 明确排除项

- Excel 第 189 行及以后“三级驾驶舱”不在本基线范围内。
- 整个“AI侧”工作表不在本基线范围内。
- 第 112 行的“AI 智能分流”子能力（包括情绪识别和自动识别反馈问题类型）不纳入；随手拍的图文/定位提交、人工分派、处置、回复、督办、查询和归档仍纳入。
- 第 173 行的“智能匹配”子能力不纳入；岗位浏览、求职登记和后续人工业务处理仍纳入。

## 状态判定

后续需求表的状态仅可为“已完成”“部分完成”或“未开始”，并且必须附实现与验收证据。实际 Java 代码、数据库迁移、可执行测试或端到端命令可以组成证据；空壳服务、`/ping`、仅有表结构或历史文档中的端到端声明均不能单独证明业务完成。开始实施时应引用稳定需求编号；完成后同步更新状态、代码位置和测试证据。

## 责任服务映射

业务数据只能归属以下可运行 Java 服务；`community-common` 仅提供共享客户端/库，`community-gateway` 仅作接入层，二者不得成为业务数据归属服务。

| 业务范围 | 责任服务 | 责任边界 |
| --- | --- | --- |
| 身份、认证、RBAC、组织/网格数据范围、隐私、审计、敏感导出、消息与系统配置 | `community-auth` | 强制数据范围、身份和权限、审计与消息收件箱/通知偏好；不拥有各业务域源数据。 |
| 人口、特殊人群、网格空间资产、车辆、资源档案、生态链、关联档案、综合查询 | `community-info` | 持有信息服务源数据，以及特殊人群走访、逾期、名单比对告警及其处置状态。 |
| 服务项目、订单、收费、履约、商品/团购、支付/退款、库存、核销 | `community-service` | 持有服务和交易源数据；负责支付回调、退款、对账、风控、订单二维码与核销。 |
| 公益资助、绿色助农、法律援助入口和点击台账 | `community-welfare` | 持有公益域数据、审批/证明材料记录和法律跳转统计。 |

## 跨服务积分与告警边界

- 积分账本由 `community-affairs` 持有。`community-service` 对订单积分抵扣、退款回退等操作，以全局业务单号幂等地调用冻结、确认、释放和回退命令，并以 outbox 或等价可靠投递、补偿机制避免跨服务双写。
- `community-info` 生成特殊人群走访、逾期和名单比对业务告警，并持有其处置状态；`community-affairs` 负责设备接入、设备告警和告警转工单；`community-auth` 负责告警消息投递。
- 附件元数据随所属业务服务保存，`community-common` 仅提供客户端能力。所有服务按既定数据范围、私有访问、短期签名、下载审计和导出权限实施；跨域待办与派单在后续需求表中必须明确实时归属或创建时范围快照。
- 法律援助外链版本与点击统计归 `community-welfare`；首页只读查询归 `community-portal`；身份、统一审计、消息收件箱、通知偏好和隐私同意归 `community-auth`。

## 全量后端需求表

通用约束：所有 API 遵循 docx §7.1 的统一响应、分页和错误码约定；涉及个人信息、支付、附件或外部系统的条目同时遵循 docx §5、§6 的加密、脱敏、最小权限、审计、TLS/签名及留存要求。业务附件元数据由表中归属服务保存，私有对象使用短期签名访问并审计下载；跨服务事件使用 outbox 或等价可靠投递、全局业务号幂等和补偿。`community-common` 和 `community-gateway` 仅列为技术支撑，绝不拥有业务数据。
